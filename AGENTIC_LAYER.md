# Agentic layer — Decisor

This repo uses the **plan → build → test** workflow from the Tactical Agentic Coding course. Development is driven by autonomous agents, not by hand. The agentic layer lives in three directories:

```
.claude/commands/   # reusable agent prompts (slash commands)
adws/               # AI Developer Workflows — orchestration scripts
specs/              # implementation specs generated by the plan step
agents/             # per-run state and logs (one folder per ADW_ID)
```

## How it works

1. You open a GitHub issue describing a feature, chore, or bug.
2. You run an ADW pointing at that issue number.
3. The ADW runs three agents in sequence:
   - **Plan** — classifies the issue, creates a branch, writes a spec to `specs/`.
   - **Build** — reads the spec and implements it, commits, pushes, opens a PR.
   - **Test** — runs typecheck + lint + build and fixes anything it can.
4. You review the PR and merge. Vercel redeploys on merge to `main`.

Every commit and PR is authored by Claude Code, with the `ADW_ID` in the commit body for traceability.

## Running an ADW

### One-shot (recommended)

```bash
uv run adws/adw_plan_build_test.py <issue_number>
```

Runs all three steps and reports the final ADW_ID and PR URL.

### Step by step (if you want more control)

```bash
uv run adws/adw_plan.py <issue_number>        # prints ADW_ID
uv run adws/adw_build.py <adw_id>             # implements, opens PR
uv run adws/adw_test.py <adw_id>              # runs tests, may amend PR
```

## Slash commands

Inside Claude Code (or via `claude -p`), you can use these directly:

| Command            | Purpose                                              |
| ------------------ | ---------------------------------------------------- |
| `/feature <issue>` | Generate a spec for a new feature                    |
| `/chore <issue>`   | Generate a spec for a chore / refactor / setup task  |
| `/bug <issue>`     | Diagnose a bug and generate a fix spec               |
| `/implement <spec>`| Implement a spec produced by one of the above        |
| `/test`            | Run typecheck/lint/build and report JSON results     |
| `/classify_issue`  | Classify an issue as feature / chore / bug           |

## Requirements

- [`uv`](https://docs.astral.sh/uv/) for the Python scripts
- [`gh`](https://cli.github.com/) authenticated against the repo
- `claude` CLI (Claude Code) on the PATH
- Node.js 20+ and npm for the app itself

## State and logs

Each ADW run is identified by an `ADW_ID`. Its state and logs live at:

```
agents/<adw_id>/
  state.json       # branch, spec path, PR URL, history
  logs/
    01-classify.log
    02-plan.log
    03-implement.log
    04-test.log
```

If something goes wrong mid-workflow, you can re-run a specific step by passing the same `ADW_ID`.
